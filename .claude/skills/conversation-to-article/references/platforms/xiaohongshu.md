# 小红书文章规范

## 平台特点

- **阅读场景**: 快速浏览，碎片化
- **用户期待**: 实用、真实、可操作
- **内容偏好**: 短平快、列表化、口语化

## 输出格式：纯文本

**重要**：小红书不支持Markdown格式，必须输出纯文本。

| 禁止使用 | 替代方案 |
|----------|----------|
| `# 标题` | 直接写标题文字 |
| `**加粗**` | 用「」或【】强调 |
| `- 列表` | 用"一、二、三"或"1. 2. 3." |
| `[链接](url)` | 直接写文字，不带链接 |

## 格式规范

### 标题

| 要素 | 规范 |
|------|------|
| 长度 | 10-20字 |
| 风格 | 口语化、有亲和力 |
| 技巧 | 竖线分隔、关键词前置 |

**标题公式**:
- 主题｜亮点：「xxx攻略｜亲测有效」
- 身份+内容：「打工人必看的xxx」
- 结果导向：「用了这个方法，xxx」

### 正文结构

```
【开头】1-2句话
- 直接说结论或亮点

【正文】300-600字
- 短段落（2-3句）
- 多用列表
- 适度分点

【结尾】1-2句话
- 互动引导
- 标签
```

### 排版建议

- 段落要短
- 善用序号（1️⃣2️⃣3️⃣ 或 一、二、三）
- 关键词可加【】突出
- 结尾加话题标签
