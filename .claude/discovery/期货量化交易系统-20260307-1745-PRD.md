# PRD: 【策略：分钟开盘战法】期货突破监控报警系统

**创建时间**: 2026-03-07
**需求提出者**: 用户
**文档状态**: 定稿
**技术方案**: TqSdk行情接入 + 纯监控报警（不下单）

---

## 一、需求理解

### 1.1 策略定义

**策略名称**: 【策略：分钟开盘战法】

**核心逻辑**:
- 时间周期: 1分钟K线
- 触发条件: 开盘后出现连续3根（含）以上上涨K线，且期间不破低（后续K线最低点 >= 前面K线最低点）
- 突破点: 首次出现"后面K线最低点 < 前面K线最低点"时确认

**计算规则**:
- 区间: 突破确认当天最高价 - 当天最低价
- 止盈价位: 区间低点 + 区间 × 止盈比例（默认50%，可调）
- 止损价位: 突破前最后一个下限 + 区间 × 止损比例（默认50%，可调）
- 参考仓位: 账户资金 × 仓位比例（默认10%，仅参考显示）

### 1.2 监控范围

- 市场: 国内期货市场（上期所、大商所、郑商所、中金所、能源中心）
- 品种: 用户自选，可多选同时监控
- 每个品种独立运行策略，互不干扰

### 1.3 报警机制

**报警渠道**:
- 电脑端: 置顶弹窗 + 提示音
- 手机端: 短信通知

**报警场景**:
1. **突破警报**: 发现突破点时立即触发（一次）
2. **止盈提醒**: 价格触及止盈价位时触发
3. **止损提醒**: 价格触及止损价位时触发
4. **未成交提醒**: 14:45若未确认买入触发
5. **强制平仓提醒**: 14:55触发收盘前平仓提醒

### 1.4 用户交互

**全自动监控**:
- 系统7×24小时全自动扫描所有选中品种
- 无需人工干预，发现机会立即报警

**人工决策交易**:
- 所有报警仅作信息提示，系统不执行任何交易
- 用户收到报警后，自行在文华财经等交易软件中手动下单
- 用户点击"已买入"按钮确认，系统进入持仓监控状态

**状态流转**:
- 【监控中】→ 发现突破 → 【已突破】（报警待买入）→ 点击已买入 → 【持仓中】→ 止盈/止损/收盘 → 【监控中】（复位继续监控）

### 1.5 时间控制

- 交易日: 跟随期货市场交易日（周一至周五，节假日除外）
- 交易时段: 日盘 + 夜盘（根据品种自动识别）
- 关键时间点:
  - 14:45: 检查突破后是否已买入，未买入则报警
  - 14:55: 持仓品种强制平仓提醒

## 二、功能规格

### 2.1 行情接入模块

**功能**: 实时获取所有监控品种的1分钟K线数据

**技术要求**:
- 使用TqSdk接入，仅需行情权限，无需交易权限
- 支持全市场品种订阅
- K线数据包含: 开盘价、最高价、最低价、收盘价、成交量
- 数据延迟 < 1秒

**异常处理**:
- 断线自动重连
- 数据缺失告警
- 开盘/收盘时段特殊处理

### 2.2 突破识别引擎

**功能**: 对每个品种实时运行突破识别算法

**算法逻辑**:
1. 维护当日K线缓存（从开盘起）
2. 识别连续上涨K线序列（至少3根，close > open）
3. 检查不破低条件（每根K线low >= 前一根K线low）
4. 检测到首次破低（当前low < 前一根low）即触发突破
5. 计算当日高低点、区间、止盈止损价位

**性能要求**:
- 单个品种处理延迟 < 100ms
- 支持同时监控10个以上品种

### 2.3 报警系统

**电脑报警**:
- 弹窗置顶显示（最前端，防止被其他窗口遮挡）
- 提示音（可自定义，支持重复播放直到用户响应）
- 弹窗内容: 策略名称、品种、突破价、区间、止盈止损价位、时间
- 操作按钮: 【已买入】【忽略】

**短信报警**:
- 使用阿里云短信服务
- 短信内容精简: 策略名、品种、突破价、止盈、止损
- 发送成功率 > 99%
- 异常时重试3次

**报警频率控制**:
- 同一突破点只报警一次
- 止盈/止损报警只触发一次
- 14:45/14:55提醒每个持仓品种单独触发

### 2.4 参数配置

**用户可调参数**:
- 监控品种列表: 多选框，支持全市场品种
- 止盈比例: 滑动条，范围10%-100%，默认50%，步进10%
- 止损比例: 滑动条，范围10%-100%，默认50%，步进10%
- 仓位比例: 滑动条，范围5%-50%，默认10%，仅参考显示
- 滑点: 输入框，默认1个最小变动价位

**系统配置**:
- TqSdk快期账号密码
- 阿里云短信AccessKey
- 短信签名和模板ID
- 报警声音文件路径

### 2.5 监控面板

**主界面显示**:
- 系统状态: 监控中/已停止
- 当前时间、交易日状态
- 监控品种列表及各自状态

**品种状态显示**:
- 品种代码和名称
- 当前状态: 监控中/已突破/持仓中
- 最新价、当日涨跌
- 突破价（如有）
- 止盈价/止损价（如有）
- 距离14:55剩余时间（持仓中状态）

**操作按钮**:
- 开始/停止监控
- 设置参数
- 查看日志
- 手动复位指定品种

### 2.6 数据持久化

**存储内容**:
- 用户配置参数
- 报警历史记录
- 当日监控状态（用于程序重启恢复）

**存储方式**: SQLite本地数据库

**日志记录**:
- 系统运行日志（启动、停止、异常）
- 行情数据日志（可选，用于复盘）
- 报警发送日志
- 用户操作日志

## 三、技术方案

### 3.1 技术栈

| 模块 | 技术选型 | 说明 |
|------|----------|------|
| 行情接入 | TqSdk Python库 | 接入快期行情，只读权限 |

### 3.3 数据流

```
TqSdk行情服务器
    ↓ 1分钟K线数据
品种监控器（每个品种独立）
    ↓ 实时计算
突破识别算法
    ↓ 触发信号
报警决策中心
    ├─→ 突破警报 ──→ 电脑弹窗 + 短信
    ├─→ 止盈止损 ──→ 价格监控器 ──→ 触及报警
```

- 突破价位
- 止盈价位
- 止损价位
- 突破时间
- 买入确认时间

**报警记录**:

## 五、风险说明

### 5.1 技术风险

- 行情延迟: TqSdk行情为Level-1快照，500ms更新一次，对1分钟策略足够
- 网络中断: 断线期间可能错过突破点，需设计重连机制

### 5.2 交易风险

⚠️ **重要声明**:
- 本系统仅为监控报警工具，不构成投资建议
- 所有交易决策由用户自行做出
- 策略过往表现不代表未来收益
- 期货交易风险巨大，可能导致本金全部损失
- 请先用模拟盘充分验证策略有效性

### 5.3 合规说明

- 本系统不执行任何交易，不涉及程序化自动交易监管
- 纯信息服务性质，合规风险极低
