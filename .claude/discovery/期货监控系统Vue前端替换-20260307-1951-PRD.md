# PRD: 期货监控系统 Vue 前端替换

## 一、需求理解（为什么）

### 1.1 目标
- 将当前 Tkinter 前端替换为**专业标准化 Vue 前端**。
- 保留并复用现有真实行情监控与策略能力（TqSdk + Breakout 策略 + 状态机 + 告警）。
- 形成可运行、可部署的前后端分离系统。

### 1.2 动机
- 现有桌面 UI 的可视化、可扩展性与标准化程度有限。
- 用户希望获得更专业的前端体验（可维护、可扩展、可部署）。

### 1.3 约束
- 数据与策略能力必须继续使用现有 Python 代码资产：
  - `futures_monitor/market.py`
  - `futures_monitor/strategy/breakout.py`
  - `futures_monitor/strategy/state_machine.py`
  - `futures_monitor/main.py`（逻辑可重构）
- 真实行情必须保留：`use_real_market_data=true` + `strict_real_mode=true`。
- ALL 模式保留（读取全部有效品种）。
- 本期要求包含部署能力（本地可运行部署，预留生产部署方式）。

### 1.4 边界
- 本期不做自动下单（仍为报警/监控系统）。
- 不引入外部交易执行通道。
- 不改变策略定义本身，仅改变交互层与系统集成方式。

### 1.5 交付物清单（最小可验证单元）
1. 后端 API 层（HTTP + 实时推送）
2. 监控控制与会话管理模块（启动/停止/标记已买入）
3. Vue 前端工程（页面、状态管理、组件）
4. 前后端联调与运行脚本
5. 部署方案与配置文档
6. 验证用例（关键接口 + 核心流程）

### 2.2 三重维度尽调

#### 模块 A：后端服务化

**深度（原理与实现）**
- 推荐：FastAPI + Uvicorn。
- 原因：
  - Python 生态成熟，和现有代码同语言，重用成本低。
  - 提供 OpenAPI、Pydantic 校验，接口标准化。
  - 支持 WebSocket，适合推送行情行更新与日志流。
- 关键实现模式：
  - 将 `MonitorApp` 的队列事件抽象为“事件总线”（row/log/connection/running）。
  - HTTP 用于命令控制（start/stop/mark-bought/config）。
  - WebSocket/SSE 用于订阅实时事件。

**广度（方案对比 ≥3）**

| 方案 | 优点 | 缺点 | 适配度 |
|---|---|---|---|
| FastAPI + WebSocket | Python统一栈、文档好、实时能力强 | 需补充连接管理 | 高 |
| Flask + Flask-SocketIO | 社区成熟 | Async模型与现有线程混用复杂 | 中 |
| Django + Channels | 完整框架 | 过重、改造成本高 | 低 |

**前瞻性（2024-2026）**
- Python 服务端正持续向 ASGI 生态集中，FastAPI + Uvicorn 是主流工程实践。
- 对于实时看板型系统，WebSocket + 前端状态流更新已是标准路径。

**结论**
- 选型：**FastAPI + WebSocket**。

---

#### 模块 B：前端应用

**深度（原理与实现）**
- 推荐：Vue 3 + Vite + TypeScript + Pinia + Element Plus。
- 原因：
  - Vue 3 在数据看板场景开发效率高，TypeScript 保证工程可维护性。
  - Element Plus 可快速搭建专业风格表格/表单/状态面板。
  - Pinia 管理实时数据和连接状态清晰。

**广度（方案对比 ≥3）**

| 方案 | 优点 | 缺点 | 适配度 |
|---|---|---|---|
| Vue3 + Vite + Element Plus | 成熟、组件完整、上手快 | 样式需统一规范 | 高 |
| React + Ant Design | 生态大 | 与用户指定 Vue 不一致 | 中 |
| Vue3 + Naive UI | 轻量现代 | 企业模板相对少 | 中 |

**前瞻性（2024-2026）**
- Vue3 + Vite + TS 已成为前端工程标准组合。
- 组件化看板与实时状态管理已趋于标准化（Store + WS 订阅）。

**结论**
- 选型：**Vue3 + Vite + TS + Pinia + Element Plus**。

---

#### 模块 C：集成与验证

**深度**
- 通信契约采用“命令 API + 事件流”双通道。
- 关键事件类型：
  - `row`（品种行状态）
  - `log`（日志）
  - `connection`（连接状态）
  - `running`（运行状态）
- 测试策略：后端单元 + 集成，前端组件 + e2e（至少关键流程）。

**广度（方案对比）**

| 方案 | 优点 | 缺点 | 适配度 |
|---|---|---|---|
| HTTP + WebSocket | 控制与推送职责清晰 | 需处理断线重连 | 高 |
| 纯轮询 HTTP | 实现简单 | 实时性差、性能浪费 | 低 |
| gRPC 双向流 | 性能强 | 前端浏览器接入复杂 | 低 |

**前瞻性**
- 浏览器实时场景中，WebSocket 仍是主流稳定方案。

**结论**
- 选型：**HTTP + WebSocket**。

---

#### 模块 D：部署

**深度**
- 本地：
  - 后端：`uvicorn app:app --host 0.0.0.0 --port 8000`
  - 前端：`vite build` 后由 Nginx 托管，反代 `/api` 与 `/ws`。
- 生产：Nginx + Systemd（或 Windows NSSM）管理后端进程。

**广度（方案对比）**

| 方案 | 优点 | 缺点 | 适配度 |
|---|---|---|---|
| 前后端分离 + Nginx 反代 | 标准化、可扩展 | 需额外部署配置 | 高 |
| 后端静态托管一体化 | 简化部署 | 扩展性一般 | 中 |
| Docker Compose | 环境一致性好 | 用户机器需Docker | 中 |

**前瞻性**
- 标准企业部署仍以反向代理 + 独立服务进程为主。

**结论**
- 选型：**前后端分离 + Nginx 反代**（同时保留本地单机快速启动）。

---

### 2.3 推荐总体方案
- 后端：FastAPI 服务化封装现有监控引擎。
- 前端：Vue3 专业看板替换 Tkinter。
- 通信：HTTP 控制 + WebSocket 推送。
- 部署：本地快速启动 + 生产 Nginx 反代。

## 三、实现路径（怎么办）

### 3.1 用户动线

1. 用户打开 Web 页面。
2. 页面初始化：读取配置、建立 WebSocket。
3. 用户编辑品种（支持 `代码 + 说明`，支持 `ALL`）并点击启动。
4. 后端启动监控线程并推送状态；前端表格实时刷新。
5. 发生突破后，日志流与行状态更新显示。
6. 用户可在前端点击“标记已买入”。
7. 用户点击停止，后端停止监控并返回已断开状态。

决策分支：
- 若配置是 ALL：后端解析为当前有效合约列表并开始订阅。
- 若 real 模式连接失败且 strict=true：立即报错，不回退 mock。

---

### 3.2 系统架构

```
[Vue 前端]
  ├─ HTTP: /api/config /api/start /api/stop /api/mark-bought
  └─ WS:   /ws/events  (row/log/connection/running)

[FastAPI 后端]
  ├─ API Router
  ├─ Monitor Service（复用现有策略引擎）
  ├─ Event Hub（队列广播）
  └─ Config Service（JSON读写）

[现有核心模块]
  ├─ market.py (TqSdk real + ALL解析)
  ├─ breakout.py / state_machine.py
  ├─ storage.py
  └─ logger.py
```

### 3.3 数据流与关键接口

#### HTTP API
- `GET /api/config`：读取当前配置。
- `PUT /api/config`：保存配置。
- `POST /api/monitor/start`：启动监控（symbols 可覆盖配置）。
- `POST /api/monitor/stop`：停止监控。
- `POST /api/monitor/mark-bought`：标记品种已买入。
- `GET /api/monitor/status`：当前运行状态与连接状态。

#### WebSocket
- `GET /ws/events`
  - 服务器推送：
    - `{"type":"row", ...}`
    - `{"type":"log", ...}`
    - `{"type":"connection", ...}`
    - `{"type":"running", ...}`

#### 安全与配置
- 快期账号密码不在前端暴露明文日志。
- 可选将密码改为环境变量覆盖（后续规划）。
